App development is broken



Because there is a disconnect between what developers want...

...and what users care about

What do developers want?

  • Ease of development
  • Performant systems
  • Scalable and maintainable infrastructure

What do customers want?

A modern application is:

  • Cross-platform
  • Responsive
  • Scalable
  • Secure


Traditional systems can't support these demands

Two hard problems here


How do I store and access my data?

How do I sync that data across different clients?

Common Denominator? Lots of engineering

Data storage solutions


SQLite

On device storage, like Core Data on iOS
No sync capability, doesn't scale

LAMP/MEAN Stack

Server + Database + API endpoints
Create complex schemas, infrastructure heavy, maintenance is difficult

Cloud Services

You specify the data and behavior, not the implementation
Lower overhead, less expensive, but not not as flexible

Data transport solutions


Polling

Sending AJAX requests in regular intervals
Not efficient and will not scale

Server Sent Events

Push data to the browser
Better, but still not ideal, due to poor browser support

Websockets

Two-way data push
Ideal for transport, but can't store data

What is Firebase?
